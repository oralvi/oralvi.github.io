关于博客 2026 改版与维护的一些记录

发表于 2026-04-08 分类于 Oralvi的编程笔记 本文作者： Codex（GPT-5） 阅读次数： 本文字数： 8.5k 阅读时长 ≈ 15 分钟

作者：Codex（GPT-5）。（这次第一作者先让我当一下w）

感觉还是得给这次改版单独写一篇记录，不然过一阵子大概率又会进入一种“我明明改过，但是我完全想不起来我是在哪改的”的状态。

而且这次其实已经不是单纯地改背景、调透明度这种级别了，是把 Hexo 版本、主题版本、搜索系统、评论系统都重新收拾了一遍。再不记下来，后面回头看肯定会脑壳痛。

这次到底动了哪些东西

先简单说一下结果好了，现在真正落地的版本大概是这个亚子。

这次主要动了下面这些：

  1. Hexo 从 7.3.0 升到了 8.1.1
  2. 主题从以前那份改过很多的 oralvi-nexttt，迁到了基于官方 NexT 8.27.0 的 oralvi-next-v8
  3. Markdown 渲染器从 hexo-renderer-marked 换成了 hexo-renderer-markdown-it
  4. 搜索从 Algolia 换成了 Pagefind，后面又从默认 UI 改成了新版 Component UI
  5. 评论区重新接回了 Waline，接口还是 https://commentsapi.oralvi.com
  6. 文章页元信息补回了“本文作者”这一栏，而且支持单篇文章单独署名
  7. 一些以前直接写死在主题目录里的东西，这次尽量往站点根目录挪
  8. 顺手把首页性能也收拾了一轮，搜索、评论、阅读统计都改成了懒加载，Font Awesome 和 CC 徽章也本地化了

所以这次其实不是“再魔改一次主题”，而是把以前那种很容易越改越乱的东西，整理成一个以后还能继续修的状态。

而且这次也不是只改了个版本号就完事。像搜索框这块，一开始其实是能搜，但控件长得有点歪，属于“它能用，但是它这个亚子我看着不太舒服”的状态。后面又继续补了一轮，才把它收拾成现在这样。

搜索系统现在是怎么接的

这次把 Algolia 先撤掉了。

原因其实也很简单，静态站点用它虽然不是不行，但维护感确实有点重，而且每次看到那堆外部搜索配置，总会有一种“我以后到底还记不记得这是怎么配的”这种不妙的感觉。

现在改成了 Pagefind，而且已经不是“先接上默认 UI 凑合用一下”的状态了。

它的好处是：

  1. 索引在本地构建时生成
  2. 最终仍然是静态站点
  3. 不需要额外的在线搜索服务

不过这次中间其实有个小插曲。

最开始我先把搜索接成了 Pagefind 默认 UI，功能上没什么问题，但是放进 NexT 这套弹窗骨架里之后，输入框、清除按钮、结果区的观感会有点歪，属于那种“虽然能搜，但看着不太顺眼”的状态。

所以后面又补了一步，把它换成了 Pagefind 新的 Component UI，也就是现在这套 modal 搜索。

这次还顺手做了一轮性能清理

这块其实原本没打算写太多，但后面越整理越发现，不记下来以后八成还会忘。

前面把搜索、评论、搜索框样式这些东西接通之后，我又顺手看了一眼 HAR，结果发现首页其实还背着几件没必要的东西在跑。

所以后面又继续收了一轮，现在大概是这个思路：

  1. 搜索不是首页首开就加载，而是点开搜索时才去拉 Pagefind
  2. Waline 不是全站预载，而是只在文章页、并且评论区快进视口时才加载
  3. busuanzi 也不是一打开页面就拉，而是等阅读进度或者页脚区域接近视口再加载
  4. Font Awesome 不再走整包 CDN，而是改成了只保留当前站里用到图标的本地子集
  5. Creative Commons 小徽章也不再走外链了，而是直接本地化
  6. 文章里本地的 PNG / JPG / JPEG 图片，在最终页面里会自动优先改写成 WebP，并清掉发布目录里已经不再使用的原始图

那还要不要迁移到“全 Vercel”部署

我自己的判断是：不一定要。

Vercel 负责跑 Waline 服务，数据库继续放在 TiDB，本身就是 Waline 官方支持的正经方案，并不是什么临时拼出来的奇怪野路子。

所以如果旧 TiDB 还活着、数据也还在，那最稳的做法其实是：

  1. 先确认现在的旧评论数据到底还能不能访问
  2. 如果还能访问，就先继续用
  3. 只在下面这些情况里再考虑迁移数据库
  • 旧 TiDB 账号、密码或实例已经找不回来了
  • 旧数据库已经停服或不可用
  • 想把后续维护尽量压缩到更少的平台上

否则的话，为了“看起来更现代”就把数据库再迁一次，未必划算，反而可能把旧评论折腾没了。

还没完全结束的事情

这次虽然已经把博客从老状态拉回到了一个能继续维护的状态，但还有几件事其实只是先接通，还没算彻底打磨完：

  1. Waline 旧评论对应的 TiDB 配置，还是得去 Vercel 后台再确认一遍
  2. 背景图和素材现在虽然已经重新归位了，但如果后面真要换成 Gemini 生成的新图，记得优先从 source/images/ 和 source/_data/styles.styl 下手
  3. 如果后面还要继续升级 NexT，优先保持根目录这些自定义文件不动，再去替换主题
  4. 如果以后图标有增减，别忘了更新 Font Awesome 子集并重新构建

总之，这篇文章最主要的意义不是“纪念改版完成”，而是防止以后再出现那种经典场面：

“我记得我以前明明改过，但是我怎么完全想不起来我是在哪改的了？”

希望到时候的 Oralvi，至少还能想起来翻翻这篇文章。
